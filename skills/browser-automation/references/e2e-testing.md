# E2E テストガイド

## テストシナリオの書き方

```
## シナリオ: カテゴリフィルタが機能する

1. navigate → http://localhost:8080
2. snapshot → サイドバーの nav 要素を確認
3. click → 「開発者」カテゴリボタン（ref 指定）
4. snapshot → スキル一覧が開発者カテゴリのみになっていることを確認
5. evaluate → フィルタ結果のカウントを検証
6. screenshot → 結果を保存
```

## 検証のコツ

- **DOM の値を検証するには `browser_evaluate`** を使う。screenshot は人間の目視確認用
- **動的コンテンツは `browser_wait_for`** で待機してから検証する
- **snapshot の `ref` は毎回変わる**。操作前に必ず最新の snapshot を取る
- **フォーム操作後は snapshot を再取得** して状態変化を確認する

## テスト戦略

### 何を E2E テストするか

E2E テストはコストが高い。以下に絞る:

1. **クリティカルユーザーフロー** — ログイン、主要機能の一連の操作
2. **統合ポイント** — フロントエンド↔API の結合部分
3. **リグレッション** — 過去に壊れた箇所

ユニットテストや統合テストで検証できるものは E2E に含めない。

### ローカル開発環境での E2E

```bash
# エミュレーター + API + フロントエンドを起動してからテスト
pnpm emulators:up
pnpm dev          # API（ポート 3000）
cd web && pnpm dev # フロントエンド（ポート 8080）
```

起動確認後に Playwright MCP でブラウザ操作を行う。
